控销专区

发表于 2019-01-25 | 分类于 技术 | 阅读次数：

1 数据表

1.1 控销专区表

表名：mph_sale_control_group，实体：SaleControlGroupEntity

字段名 描述
scg_id 主键
scg_name 控销/诊所专区名称
scg_desc 描述
scg_sort 排序
scg_type 专区类型：1控销，2诊所
scg_status 状态：-1已删除， 0无效， 1生效
su_id 供应商ID
goods_count 商品数量

一条主表信息对应供应商的一个控销专区，但当设置商品少于4个，统一显示在系统默认的“单品专供控销区”

1.2 控销商品组

表名：mph_sale_control_goods_group，实体：SaleControlGoodsGroupEntity

字段名 描述
scgg_id 主键
scg_id 控销专区ID
scgg_sort 商品组排序

一个控销专区可以对应多个控销商品组，一个商品组信息关联多个商品和多个客户信息

1.3 控销商品信息

表名：mph_sale_control_goods，实体：SaleControlGoods

字段名 描述
scg_id 控销专区ID
scgg_id 商品组ID
g_id 商品ID
scgg_price 商品配置价格
scgg_sort 商品排序

同一个控销专区下的商品分组的商品ID一致，只是scgg_price不同

1.4 控销客户信息表

表名：mph_sale_control_goods，实体：SaleControlGoods

字段名 描述
scgc_id 主键ID
scgg_id 商品组ID
scg_id 控销专区ID
scgc_type 类型 1客户ID， 2业务类型， 3销售区域
scgc_type_id 客户ID

按业务分组、区域分组、指定客户三种类型进行分类

1.5 控销申请记录表

表名：mph_sale_control_apply，实体：SaleControlApply

字段名 描述
sca_id 主键ID
scg_id 控销专区ID
g_id 商品id
m_id 申请账号id
e_id 企业id
su_id 供应商id
sca_contractor 联系人
sca_mobile 联系人手机号
sca_area 申请区域，申请人填写
sca_sale_num 月销售量，申请人填写
sca_reason 申请理由
sca_audit_reason 未通过的理由
sca_state 状态 0 未通过 1 已通过 2 未处理
sca_add_time 申请时间
sca_audit_time 审核时间
audit_refuse_reason 1该控销区客户已满建 2议申请其他区域 3其他
audit_refuse_remark 不通过原因描述

多次申请只生成一个记录

2 接口列表

2.1 查询供应商销控专区列表

/sales/control/sale_group_main.html

查询信息包含：

  • 商家控销专区列表展示（包含专区描述信息和所有商品名称）
  • 搜索条件：全部商品组
  • 用户专销权限申请个数

2.2 新增或者修改控销专区信息

/sales/control/control_group_save.html

新增步骤：

  1. 查询供应商正在参与促销的商品ID列表，判断控销专区中的商品是否与之重合，如果是，则添加失败并返回正在参与活动的商品名称,更新商品索引
  2. 新增或修改
    1）如果是修改，必须先清空专区旧的数据，步骤：
    ①获取控销专区商品组列表；
    ②删除每个商品组关联的客户信息表；
    ③删除每个商品组关联的商品定价信息表，并且还原这些商品在商品表中的g_type和g_tag字段；
    ④删除控销专区商品表数据；
    2）新增或修改控销专区表信息
    3）添加商品组表信息
    4）添加商品定价表信息并更新商品的g_type和g_tag字段并刷新商品索引
    5）添加商品组客户信息

2.3 控销权限申请

/goods/applySaleControl

  1. 根据suId和gid查询控销专区信息，如果没有，则返回提示
  2. 根据scgId,suId,eid查询用户是否已经申请,分三种情况：
    ①已申请，且申请通过
    ②已有申请，审核未通过（更新申请时间）
    ③未通过申请,新增一条申请记录
  3. 新增一条申请记录，申请状态设置为0（待处理状态）

2.3 控销权限审批

/sales/sale_control/update_sale_control_apply_status.html
用户在前台需要选中一个商品组或新增商品组

  1. 如果选中不通过，则将拒绝原因更新到申请表
  2. 如果通过，同样跟新申请表状态，并在控销专区客户表中新增一条信息，成功后用户就能够拥有这个控销专区的购买权限，并享受选中的用户组的商品控销价。
